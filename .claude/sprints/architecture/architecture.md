## Overview

**Name**: Lando DS (`@lando-labs/lando-ds`)
**Purpose**: A self-describing, AI-native React component library — production-ready components, semantic OKLCH design tokens, a headless hooks library, and a structured `meta.json` artifact an AI assistant grounds itself against.
**Primary users**: React/Next.js app teams, and the AI coding agents working alongside them.
**Shape**: Library-only. No in-repo showcase or dev server — components are exercised through their test suites and verified by consuming the built package from a host app (`examples/next-app-router`).

## Shared / Core Areas

Cross-cutting concerns affecting multiple feature areas.

| Area | Primary files | Role |
|---|---|---|
| **Design tokens** | `src/tokens/*`, `src/styles/tokens.css` | OKLCH semantic tokens (`--color-primary`, …); the single theming surface. Generated by `scripts/emit-tokens.mjs`. |
| **Theming runtime** | `src/utils/ThemeProvider.tsx`, `themeScript.ts` | `ThemeProvider` + `useTheme`, dark mode via `[data-theme]`, SSR-safe theme init |
| **Hooks library** | `src/hooks/*` (19 hooks) | Headless, dependency-free, `'use client'`, SSR-safe; importable from root or `/hooks` |
| **Self-describing meta** | `scripts/emit-meta.mjs` → `dist/meta.json` | Per-component prop tables, capabilities, hooks, tokens, presets — the AI-grounding artifact |
| **Icon system** | `src/components/Icon/*`, `src/icons.ts` | `Icon` wrapper over a curated `lucide-react` set |

## Discipline Areas

For global issues that span features, group by discipline.

| Discipline | Scope | Enforcement |
|---|---|---|
| **Accessibility** | ARIA, keyboard nav, focus mgmt, contrast (WCAG 2.1 AA target) | tests + `jest-axe` |
| **RSC-safety** | server-safe vs client-only boundaries | build + `meta.capabilities`, `reference/rsc-boundary-matrix.md` |
| **Customizability** | tokens, `className`/`style` passthrough, cascade layers, `as`/`asChild` | `reference/css-layers.md`, `composition-contract.md` |
| **State API** | uncontrolled-first (`defaultX` + `X`/`onXChange`) | `npm run validate:state` |
| **Testing** | unit coverage (96/96 component dirs have ≥1 test) | Vitest; CI gate |
| **Performance** | bundle size, tree-shaking, per-component specifiers | subpath exports + shims |
| **Docs / claims** | verify-before-claim; generated inventories | `npm run sync-docs:check` |

## Architecture Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Styling | **CSS Modules + design tokens** (not Tailwind) | Scoped styles, explicit token usage, clean component boundaries |
| Color system | **OKLCH semantic tokens** | Perceptually-uniform, re-skinnable by overriding aliases alone |
| Self-description | **`meta.json` as source of truth** | AI agents/MCP ground on the real API surface, not guesses |
| CSS delivery | **Cascade layers** (`@layer`) | Consumer styles win predictably; documented unlayered escape hatch |
| Rendering | **RSC-native** | Server-safe components annotated + enforced through the build |
| State | **Uncontrolled-first** via `useControllableState` | Simple case works out of the box; controlled available |
| Composition | **`as` (primitives) + `asChild`/`Slot` (universal)** | Radix-style polymorphism, grounded in `meta.polymorphic` |
| Dist policy | **`dist/` not committed** (rebuilt on demand) | Avoids stale partial trees; tarball smoke test is the CI guarantee |

## Build & Distribution

| Field | Value |
|---|---|
| **Runtime** | React 19 (18 supported via peer dep), TypeScript 5, Node ≥20 |
| **Build** | Vite library mode (`vite.config.ts`) + `vite-plugin-dts`; `scripts/emit-*.mjs` for tokens, meta, unlayered CSS, theme-init, component shims |
| **Outputs** | ESM (`index.js`), CJS (`index.cjs`), `.d.ts`, `design-system.css`, `meta.json` |
| **Exports** | `.`, `./components`, `./tokens`, `./hooks`, `./styles`, `./meta`, `./icons`, … |
| **Runtime deps** | `lucide-react` (peer), `recharts`, `react-markdown` + `rehype-sanitize` (Markdown) |
| **Distribution** | Public npm as `@lando-labs/lando-ds` via Changesets (`.github/workflows/release.yml`), published from CI with provenance |
| **Gate** | typecheck → lint → test → build, plus `validate:meta`/`state`/`hooks` + a packed-tarball type-resolution smoke test (`examples/consumer-smoke`) |
