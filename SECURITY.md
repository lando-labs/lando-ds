# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities responsibly. **Do not open a public issue for a security problem.**

Instead, open a private report via GitHub: go to the repository's **Security** tab → **Report a vulnerability** (GitHub Security Advisories).

Please include a description of the issue, the affected version(s), reproduction steps, and the potential impact. We aim to acknowledge reports within a few business days and will keep you updated as we investigate and prepare a fix.

## Supported versions

Security fixes are released against the latest published `@lando-labs/lando-ds` version. We recommend staying current with the most recent release.

## Scope

This policy covers the published `@lando-labs/lando-ds` package and the code in this repository. Because some components render user-provided content (e.g. `Markdown`, `CodeBlock`, chat surfaces), reports about sanitization, escaping, or injection surfaces are especially welcome.
